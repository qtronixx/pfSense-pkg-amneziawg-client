# pfSense-pkg-amneziawg-client

## Структура проекта

```
pfSense-pkg-amneziawg-client/
├── Makefile                              # Сборочный файл для FreeBSD pkg
├── pkg-plist                             # Список файлов, входящих в пакет
├── pkg-descr                             # Описание пакета для pkg-менеджера
├── files/
│   ├── usr/local/pkg/
│   │   ├── awg.inc                       # Основная backend-библиотека (PHP)
│   │   ├── awg.xml                       # XML-описание пакета для Package Manager
│   │   └── awg_validate.inc              # Валидация полей веб-форм
│   ├── usr/local/www/
│   │   ├── vpn_awg_tunnels.php           # Страница списка клиентских туннелей
│   │   ├── vpn_awg_edit.php              # Форма создания/редактирования клиента
│   │   ├── vpn_awg_status.php            # Страница статуса (аналог `awg show`)
│   │   └── widgets/widgets/
│   │       └── awg_status.widget.php     # Виджет для панели управления
│   ├── usr/local/etc/rc.d/
│   │   └── awg.sh                        # rc.d-скрипт управления службой
│   ├── etc/inc/priv/
│   │   └── awg.priv.inc                  # Определение привилегий для WebGUI
│   └── usr/local/share/pfSense-pkg-amneziawg-client/
│       └── info.xml                      # Метаинформация пакета (версия, зависимости)
└── scripts/
    └── build_kmod.sh                     # Скрипт сборки модуля ядра if_awg.ko из исходников
```

## Описание компонентов

### Сборочные файлы
| Файл | Назначение |
|------|------------|
| `Makefile` | Определяет процесс сборки FreeBSD-пакета, зависимости и этапы установки |
| `pkg-plist` | Перечисляет все файлы, которые будут включены в пакет |
| `pkg-descr` | Краткое описание плагина, отображаемое в pkg-менеджере |

### Backend (PHP-библиотеки)
| Файл | Назначение |
|------|------------|
| `awg.inc` | Основная библиотека с функциями управления туннелями, конфигурацией и взаимодействия с демоном AmneziaWG |
| `awg.xml` | XML-конфигурация для интеграции с Package Manager pfSense (пункты меню, сервисы, зависимости) |
| `awg_validate.inc` | Функции валидации пользовательского ввода для всех форм плагина |

### WebGUI (интерфейс)
| Файл | Назначение |
|------|------------|
| `vpn_awg_tunnels.php` | Основная страница со списком всех клиентских туннелей и действиями (добавить, редактировать, удалить) |
| `vpn_awg_edit.php` | Форма для создания нового или редактирования существующего клиентского подключения |
| `vpn_awg_status.php` | Отображение текущего статуса туннелей, информации о пирах и трафике |
| `awg_status.widget.php` | Виджет для панели управления pfSense с краткой сводкой по AmneziaWG |

### Служебные файлы
| Файл | Назначение |
|------|------------|
| `awg.sh` | Скрипт инициализации rc.d для запуска/остановки/перезапуска службы AmneziaWG |
| `awg.priv.inc` | Определяет права доступа к страницам плагина в системе привилегий pfSense |
| `info.xml` | Метаданные пакета: версия, автор, зависимости, совместимость |

### Сборка модуля ядра
| Файл | Назначение |
|------|------------|
| `build_kmod.sh` | Автоматизирует загрузку исходного кода и компиляцию `if_awg.ko` для текущего ядра FreeBSD |

## Принцип размещения файлов

Структура `files/` зеркально отражает корневую файловую систему FreeBSD:
- `files/usr/local/pkg/` → `/usr/local/pkg/` (библиотеки pfSense)
- `files/usr/local/www/` → `/usr/local/www/` (веб-интерфейс)
- `files/usr/local/etc/rc.d/` → `/usr/local/etc/rc.d/` (служебные скрипты)
- `files/etc/inc/priv/` → `/etc/inc/priv/` (привилегии)
- `files/usr/local/share/` → `/usr/local/share/` (данные пакета)
```

Пояснение по совместимости с 2.8.1

Все обращения к конфигу идут через обёртки awg_config_get_path() / awg_config_set_path() — pfSense 2.8.x постепенно переводит внутренний API на config_get_path()/config_set_path() вместо прямых обращений к $config[...]; обёртка даёт совместимость в обе стороны.
declare(strict_types=1) и явные приведения типов ((int), (string)) — под PHP 8.2, где неявные приведения null→string в некоторых контекстах генерируют Deprecated-предупреждения.
Служба зарегистрирована как нормальный rc.d-сервис с REQUIRE: NETWORKING, а не через костыль Shellcmd/earlyshellcmd, как в оригинальной инструкции — это убирает лишнюю пакетную зависимость и решает ту же проблему порядка загрузки штатно.
